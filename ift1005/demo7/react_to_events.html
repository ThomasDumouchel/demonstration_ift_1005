Réagir à des événements

Les fichiers script sont surtout intéressant parce qu'ils permettent de réagir à des événements, tel que

  • click: quand l'utilisateur clique sur un élément
  • mouseover: quand la souris passe au-dessus d'un élément
  • mouseout: quand la souris sort d'un élément
  • keydown: quand une touche du clavier est pressée
  • keyup: quand une touche du clavier est relâchée
  • change: quand le contenu d'un élément form change de valeur
Voici une liste exhaustive: https://www.w3schools.com/jsref/dom_obj_event.asp

On peut réagir à ces événements en utilisant la méthode addEventListener() de l'objet window.
Cette méthode prend deux paramètres:

  • Le nom de l'événement
  • Une fonction qui sera appelée quand l'événement se produit
Exemple: 
                window.addEventListener("click", function() {
                    alert("Vous avez cliqué!");
                });

On peut également réagir à des événements en utilisant des attributs HTML.
Par exemple, on peut réagir à un click sur un bouton en utilisant l'attribut onclick. 

                <button onclick="alert('Vous avez cliqué!')">Cliquez-moi</button>

Ou même appeler une méthode définie dans un script directement: 
                <button onclick="myFunction()">Cliquez-moi</button>

Exercises:

  • Ajouter un bouton qui affiche un message quand on clique dessus
  • countClicks: Créer un bouton qui affiche le nombre de fois qu'il a été cliqué.
  • live form validation: Créer un formulaire avec un champ de texte qui refuse la lettre b
  • inject products: injecter des produits dans la page à partir d'un fichier JSON
  • Faire un carré qui s'échappe de mon curseur
